<h2>Core Concept</h2>
<p>Formdown uses a simple pattern: <code>@field_name: [type attributes]</code></p>
<pre class="language-markup"><code class="language-markup">@name: [text required]
@email: [email required]
@submit: [submit label="Send"]</code></pre><p><strong>That&#39;s it!</strong> This creates a complete form with validation.</p>
<h2>Field Anatomy</h2>

</tr>
</thead>
<tbody><tr>

<td>✅</td>

</tr>

</tr>

<p>Override automatic labels when needed:</p>

<h2>Tips</h2>
<h3>Field Naming</h3>
<ul>
<li>Use descriptive names: <code>email_address</code> not <code>email</code></li>
<li>Use snake_case for consistency: <code>first_name</code> not <code>firstname</code></li>
<li>Avoid starting with numbers: <code>user_1</code> not <code>1st_user</code></li>
</ul>
<h3>Organization</h3>
<ul>
<li>Group related fields together</li>
<li>Use headers (<code>#</code>, <code>##</code>) to organize sections</li>
<li>Add descriptive text between field groups</li>
</ul>
<h3>Validation</h3>
<ul>
<li>Always mark required fields</li>
<li>Use appropriate input types (<code>email</code>, <code>tel</code>, <code>number</code>)</li>
<li>Add reasonable constraints (<code>min</code>, <code>max</code>, <code>maxlength</code>)</li>
</ul>
<h3>Labels</h3>
<ul>
<li>Let smart generation handle simple cases</li>
<li>Use custom labels for clarity when needed</li>
<li>Keep labels concise but descriptive</li>
</ul>
