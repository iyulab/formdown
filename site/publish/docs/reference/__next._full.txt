<h3>Field Naming</h3>
<ul>
<li>Use <code>snake_case</code> for consistency: <code>first_name</code>, <code>email_address</code></li>
<li>Avoid starting with numbers: <code>user_1</code> not <code>1st_user</code></li>
<li>Be descriptive: <code>phone_number</code> not <code>phone</code></li>
</ul>
<h3>Label Strategy</h3>
<ul>
<li>Use parentheses for simple custom labels: <code>@field(Label):</code></li>
<li>Use <code>label=</code> attribute for complex labels with quotes</li>
<li>Let smart generation handle simple cases: <code>@first_name: []</code></li>
</ul>
<h3>Validation</h3>
<ul>
<li>Always mark required fields: <code>@field*:</code> or <code>required</code></li>
<li>Use appropriate input types: <code>@[]</code> for email, <code>#[]</code> for numbers</li>
<li>Add constraints: <code>min</code>, <code>max</code>, <code>pattern</code> as needed</li>
</ul>
<h3>Shorthand Usage</h3>
<ul>
<li>Use shorthand for common patterns: <code>@email*: @[]</code></li>
<li>Mix shorthand and full syntax as needed</li>
<li>Prioritize readability over brevity</li>
</ul>
