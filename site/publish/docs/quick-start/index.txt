<h2>Installation</h2>
<p>Install the core package for form rendering:</p>
<pre class="language-bash"><code class="language-bash"><span class="token function">npm</span> <span class="token function">install</span> @formdown/ui</code></pre><p>For development with the editor:</p>

<h3>Creating Your First Form</h3>
<p>Create a <code>.fd</code> file with simple form fields:</p>
<pre class="language-markup"><code class="language-markup"># Contact Form

@name*: []
@email*: @[]
@message: [textarea]</code></pre><p><strong>That&#39;s it!</strong> FormDown automatically:</p>
<ul>
<li>Generates labels (&quot;Name&quot;, &quot;Email&quot;, &quot;Message&quot;)</li>
<li>Makes fields required with <code>*</code></li>
<li>Sets appropriate input types (<code>@</code> = email)</li>
</ul>
<h3>Alternative: Standard Syntax</h3>
<p>You can also use the full syntax for more control:</p>
<pre class="language-markup"><code class="language-markup"># Contact Form

@name(Name): [text required]
@email(Email Address): [email required]
@message(Message): [textarea]</code></pre><h3>Rendering with Web Components</h3>

<p>FormDown offers two ways to write forms:</p>
<h3>Shorthand (Recommended for Beginners)</h3>
<pre class="language-markup"><code class="language-markup"># Quick Registration

@username*: []
@email*: @[]
@password*: ?[]
@age: #[min=13]
@country{USA,Canada,UK}: s[]
@interests{Web,Mobile,AI}: c[]
@terms*: c[]</code></pre><p><strong>Benefits:</strong></p>
<ul>
<li>✅ <strong>Concise</strong> - Less typing, faster development</li>
<li>✅ <strong>Intuitive</strong> - Common patterns use familiar symbols</li>
<li>✅ <strong>Smart</strong> - Automatic label generation</li>
</ul>
<h3>Standard (Full Control)</h3>
<pre class="language-markup"><code class="language-markup"># Detailed Registration

@username: [text required minlength=4 placeholder="Username"]
@email: [email required]
@password: [password required minlength=8]
@age: [number min=13 max=120]
@country: [select options="USA,Canada,UK"]
@interests: [checkbox options="Web,Mobile,AI"]
@terms: [checkbox required label="I agree to terms"]</code></pre><p><strong>Benefits:</strong></p>
<ul>
<li>🔧 <strong>Flexible</strong> - Custom attributes and validation</li>
<li>🎨 <strong>Styling</strong> - CSS classes and data attributes</li>
<li>📋 <strong>Complex</strong> - Advanced form logic</li>
</ul>
<h3>Mixed Usage (Best Practice)</h3>
<pre class="language-markup"><code class="language-markup"># Registration Form

// Simple fields - use shorthand
@name*: []
@email*: @[]
@phone{(###)###-####}: []

// Complex fields - use standard
@bio: [textarea rows=4 maxlength=500 placeholder="Tell us about yourself..."]
@profile_pic: [file accept="image/*" class="upload-field"]</code></pre><h2>Next Steps</h2>
