<h2>Core Principles</h2>
<ol>
<li><strong>Full Compatibility</strong>: All shorthand syntax is fully convertible to <code>[attributes]</code> syntax</li>
<li><strong>Type-based Interpretation</strong>: <code>{content}</code> is interpreted appropriately by field type</li>
<li><strong>Additional Validation</strong>: Complex validation uses standard HTML5 attributes</li>
<li><strong>Sugar Syntax</strong>: Provides convenience without limiting functionality</li>
</ol>
<h2>Shorthand Markers</h2>
<h3>Required Marker</h3>
<ul>
<li><code>*</code> - Placed after field name</li>
</ul>
<h3>Type Markers (before brackets)</h3>
<ul>
<li><code>@</code> = email</li>
<li><code>#</code> = number  </li>
<li><code>***</code> = password</li>
<li><code>http://</code> = url</li>
<li><code>tel:</code> = tel</li>
<li><code>date:</code> = date</li>
<li><code>time:</code> = time</li>
<li><code>dt</code> = datetime-local</li>
<li><code>M</code> = month</li>
<li><code>W</code> = week</li>
<li><code>$</code> = number (money input with currency formatting)</li>
<li><code>T</code> = textarea (+ number for rows, e.g., T5)</li>
<li><code>F</code> = file</li>
<li><code>C</code> = color</li>
<li><code>R</code> = range</li>
<li><code>r</code> = radio</li>
<li><code>s</code> = select</li>
<li><code>c</code> = checkbox</li>
</ul>
<h3>Content Definition (<code>{content}</code>)</h3>
<p><strong>Core Principle</strong>: <code>{content}</code> is interpreted by type</p>
<ul>
<li><strong>Text-based types</strong> → <code>pattern</code> attribute (regex validation)</li>
<li><strong>Date/time types</strong> → <code>format</code> attribute (format specification)</li>
<li><strong>Selection types</strong> → <code>options</code> attribute (choice options)</li>
</ul>
<h3>Label Definition</h3>
<ul>
<li><code>(Label Text)</code> = [label=&quot;Label Text&quot;]</li>
</ul>
<h3>Inline Notation</h3>
<ul>
<li><code>___@fieldname</code> = inline field</li>
</ul>

<h3>1. Use Shorthand for Common Patterns</h3>
<pre class="language-markup"><code class="language-markup">// Good - Clear and concise
@email*: @[]
@age: #[min=0 max=120]
@message*: T5[]

// Avoid mixing when standard is clearer
@complex_field: [text data-custom="value" aria-label="Complex"]</code></pre><h3>2. Pattern Validation</h3>
<pre class="language-markup"><code class="language-markup">// Good - Clear pattern in field name
@username{^[a-zA-Z0-9_]{3,20}$}: []
@zip_code{^[0-9]{5}$}: []

// For complex patterns, consider standard syntax
@field: [pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}"]</code></pre><h3>3. Keep Options Readable</h3>
<pre class="language-markup"><code class="language-markup">// Good - Pipe separator for few options
@size: r{S|M|L|XL}[]

// Good - Comma for many options
@country: s{USA,Canada,Mexico,UK,France,Germany,Japan,Australia}[]</code></pre><h3>4. Smart Label Usage</h3>
<pre class="language-markup"><code class="language-markup">// Let FormDown generate labels from good field names

@email_address*: @[]       // → "Email Address"
