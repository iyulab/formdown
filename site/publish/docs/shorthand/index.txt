<h2>Why Shorthand?</h2>
<p>Compare these equivalent forms:</p>
<h3>Standard Syntax</h3>
<pre class="language-markup"><code class="language-markup">@name: [text required]
@email: [email required]

<pre class="language-markup"><code class="language-markup">@name*: []

<h2>When to Use Shorthand</h2>
<h3>✅ Perfect for Shorthand</h3>
<ul>
<li><strong>Common field types</strong>: email, phone, numbers</li>
<li><strong>Simple validation</strong>: required fields, basic patterns</li>
<li><strong>Rapid prototyping</strong>: quick form mockups</li>
<li><strong>Standard forms</strong>: contact, registration, surveys</li>
</ul>
<h3>🔧 Use Standard Syntax</h3>
<ul>
<li><strong>Complex validation</strong>: multi-step regex patterns</li>
<li><strong>Custom attributes</strong>: CSS classes, data attributes</li>
<li><strong>Framework integration</strong>: dynamic values</li>
<li><strong>Advanced features</strong>: conditional logic</li>
</ul>
