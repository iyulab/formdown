<h2>Core Principle</h2>
<p>FormDown fields use the pattern <code>@field_name: [type attributes]</code> for clarity and Markdown compatibility.</p>
<h2>Smart Label Generation</h2>
<p>FormDown automatically generates human-readable labels from field names when no custom label is provided. This feature helps create clean, readable forms without requiring explicit labels for every field.</p>
<h3>Field Naming Rules</h3>
<ul>
<li>Field names must start with a letter (a-z, A-Z)</li>
<li>Field names cannot start with a number</li>
<li>Field names can contain letters, numbers, and underscores</li>
</ul>
<h3>Automatic Label Formatting</h3>
<p>FormDown intelligently converts field names to proper labels:</p>
<pre class="language-markup"><code class="language-markup">// Snake case → Title Case
@first_name: []           → "First Name"
@email_address: []        → "Email Address"
@phone_number: []         → "Phone Number"

// Camel case → Title Case
@firstName: []            → "First Name"
@emailAddress: []         → "Email Address"
@userPhoneNumber: []      → "User Phone Number"

// Single words → Capitalized
@name: []                 → "Name"
@email: []                → "Email"
@age: []                  → "Age"</code></pre><p><strong>Three ways to define field labels:</strong></p>
<pre class="language-markup"><code class="language-markup">// Method 1: Using parentheses (recommended for readability)
@field_name(Custom Label): [type attributes]

// Method 2: Using label attribute 
@field_name: [type label="Custom Label" attributes]

// Method 3: Let FormDown generate smart labels automatically
@field_name: [type attributes]

// Examples
@user_name(Full Name): [text required]           // Custom label via parentheses
@user_name: [text required label="Full Name"]    // Custom label via attribute
@user_name: [text required]                      // Smart label: "User Name"</code></pre><p>All methods are equivalent and produce the same result when using custom labels.</p>
<h2>Label Definition Methods</h2>
<h3>Method 1: Parentheses Syntax (Recommended)</h3>
<pre class="language-markup"><code class="language-markup">@user_name(Full Name): [text required]
@email_address(Email Address): [email required]
@phone_number(Phone): [tel]</code></pre><p><strong>Benefits:</strong></p>
<ul>
<li><strong>Cleaner syntax</strong> - Separates label from attributes</li>
<li><strong>Better readability</strong> - Label stands out visually</li>
<li><strong>Less verbose</strong> - No need for <code>label=</code> attribute</li>
</ul>
<h3>Method 2: Label Attribute</h3>
<pre class="language-markup"><code class="language-markup">@user_name: [text required label="Full Name"]
@email_address: [email required label="Email Address"]
@phone_number: [tel label="Phone"]</code></pre><p><strong>Benefits:</strong></p>
<ul>
<li><strong>Attribute consistency</strong> - Treats label like any other attribute</li>
<li><strong>Familiar pattern</strong> - Similar to HTML attributes</li>
<li><strong>Conditional labels</strong> - Can use dynamic values in frameworks</li>
</ul>
<h3>When to Use Which?</h3>
<p><strong>Use parentheses <code>@field(Label):</code> when:</strong></p>
<ul>
<li>Writing static forms with fixed labels</li>
<li>Prioritizing readability and clean syntax</li>
<li>Labels are simple text without special characters</li>
</ul>
<p><strong>Use <code>label=</code> attribute when:</strong></p>
<ul>
<li>Labels contain quotes or special characters</li>
<li>Building dynamic forms with template engines</li>
<li>Consistency with other attribute patterns is important</li>
</ul>
<p><strong>Mixed usage is perfectly fine:</strong></p>
<pre class="language-markup"><code class="language-markup">@name(Full Name): [text required]
@description: [textarea label="Tell us about yourself" rows=4]
@submit_btn: [submit label="Send Application"]</code></pre><h2>Minimal Syntax</h2>
<p>FormDown supports minimal syntax for common use cases:</p>
<h3>Block Fields - Empty Brackets</h3>
<pre class="language-markup"><code class="language-markup">// Minimal syntax (defaults to text input)
@name: []
@email: []
@phone: []

// Equivalent to
@name: [text]
@email: [text] 
@phone: [text]</code></pre><h3>Inline Fields - Optional Brackets</h3>
<pre class="language-markup"><code class="language-markup">// Minimal inline syntax
Hello ___@name!
Your age is ___@age.

// Equivalent to
Hello ___@name[text]!
Your age is ___@age[text].</code></pre><p><strong>When to use minimal syntax:</strong></p>
<ul>
<li>✅ <strong>Simple text inputs</strong> without validation</li>
<li>✅ <strong>Rapid prototyping</strong> and quick forms</li>
<li>✅ <strong>Default behavior</strong> is sufficient</li>
</ul>
<p><strong>When to use full syntax:</strong></p>
<ul>
<li>🔧 <strong>Specific input types</strong> (email, number, date, etc.)</li>
<li>🛡️ <strong>Validation rules</strong> (required, min/max, pattern)</li>
<li>🎨 <strong>Custom attributes</strong> (classes, data attributes)</li>
</ul>
<h2>Field Types</h2>
<h3>Text Inputs</h3>
<pre class="language-markup"><code class="language-markup">@name: [text required]
@email: [email required]
@password: [password minlength=8]
@phone: [tel]
@website: [url]
@age: [number min=18 max=100]
@birth_date: [date]
@appointment_time: [time]

// Minimal syntax - defaults to text input
@name: []
@description: []</code></pre><h3>Text Areas</h3>
<pre class="language-markup"><code class="language-markup">@description: [textarea rows=4]
@comments: [textarea required placeholder="Your thoughts..."]</code></pre><h3>Selection Fields</h3>
<pre class="language-markup"><code class="language-markup">// Radio buttons - single selection (inline by default)
@gender: [radio options="Male,Female,Other"]

// Single checkbox - for yes/no or agreement
@newsletter(Subscribe to newsletter): [checkbox]
@terms(I agree to terms and conditions): [checkbox required]

// Checkbox group - multiple selection (inline by default)
@interests: [checkbox options="Web,Mobile,AI,Design"]

// Vertical layout for longer option lists
@preferences: [checkbox layout="vertical" options="Email Notifications,SMS Alerts,Phone Calls,Push Notifications"]

// Select dropdown
@country: [select options="USA,Canada,UK,Other"]</code></pre><p><strong>Checkbox Types:</strong></p>
<ul>
<li><strong>Single checkbox</strong>: <code>[checkbox]</code> - Creates a single checkbox for yes/no, agreement, or subscription</li>
<li><strong>Checkbox group</strong>: <code>[checkbox options=&quot;...&quot;]</code> - Creates multiple checkboxes for multi-select</li>
</ul>
<p><strong>Layout Options:</strong></p>
<ul>
<li><strong>Default (inline)</strong>: Options are displayed horizontally, wrapping to new lines as needed</li>
<li><strong><code>layout=&quot;vertical&quot;</code></strong>: Options are displayed vertically, each on its own line</li>
</ul>
<h3>Actions</h3>
<pre class="language-markup"><code class="language-markup">@submit_btn: [submit label="Submit Form"]
@reset_btn: [reset label="Clear Form"]</code></pre><h2>Inline Fields</h2>
<p>For embedding fields within text flow, use the <code>___@field[type attributes]</code> pattern:</p>
<pre class="language-markup"><code class="language-markup">Welcome back, ___@user_name[text required]!

Your order details:
- Quantity: ___@quantity[number min=1 max=100] items
- Delivery date: ___@delivery_date[date required]
- Total amount: $___@amount[number min=0 step=0.01]

Special instructions: ___@notes[textarea rows=2 placeholder="Optional notes"]

Contact preference: ___@contact_method[radio options="Email,Phone,SMS"]

// Minimal inline syntax - brackets optional for simple cases
Welcome ___@user_name!
Please enter ___@age and ___@email.</code></pre><p><strong>Benefits of <code>___</code> prefix:</strong></p>
<ul>
<li><strong>Consistency</strong>: Uses same <code>@field[type]</code> syntax as block fields</li>
<li><strong>Visibility</strong>: Easy to spot inline fields in text</li>
<li><strong>No conflicts</strong>: Unlikely to clash with existing Markdown syntax</li>
<li><strong>Parsing simplicity</strong>: Clear delimiter for parsers</li>
<li><strong>Minimal syntax</strong>: Brackets can be omitted for simple text fields</li>
</ul>
<h2>Attributes</h2>
<h3>Universal Attributes</h3>
<ul>
<li><code>label=&quot;text&quot;</code> - Custom field label (overrides field name)</li>
<li><code>required</code> - Field is mandatory</li>
<li><code>placeholder=&quot;text&quot;</code> - Placeholder text</li>
<li><code>value=&quot;default&quot;</code> - Default value</li>
<li><code>disabled</code> - Disable field</li>
</ul>
<h3>Validation Attributes</h3>
<ul>
<li><code>minlength=n</code> / <code>maxlength=n</code> - Text length constraints</li>
<li><code>min=n</code> / <code>max=n</code> - Numeric/date constraints</li>
<li><code>pattern=&quot;regex&quot;</code> - Custom validation pattern</li>
</ul>
<h3>Selection Attributes</h3>
<ul>
<li><code>options=&quot;item1,item2,item3&quot;</code> - Available choices</li>
</ul>
<h3>Text Area Attributes</h3>
<ul>
<li><code>rows=n</code> - Number of visible rows</li>
</ul>
<h3>HTML Extensibility</h3>
<p><strong>All HTML attributes are supported</strong> - FormDown passes through any attributes you specify:</p>
<pre class="language-markup"><code class="language-markup">// Standard HTML attributes
@email: [email required autocomplete="email" spellcheck="false"]
@phone: [tel maxlength=15 inputmode="tel"]
@search: [text autocomplete="off" aria-label="Search products"]

// CSS classes and styling
@username: [text class="form-control" style="border: 2px solid blue"]
@password: [password data-strength="true" aria-describedby="pwd-help"]

// Custom data attributes for JavaScript
@slider: [range min=0 max=100 step=5 data-unit="%" data-live-update="true"]
@upload: [file accept="image/*" data-max-size="5MB" data-preview="true"]

// Accessibility attributes
@bio: [textarea rows=4 aria-required="true" aria-describedby="bio-hint"]</code></pre><p><strong>Attribute Processing:</strong></p>
<ul>
<li>Boolean attributes: <code>required</code>, <code>disabled</code>, <code>readonly</code></li>
<li>String attributes: Quoted values <code>placeholder=&quot;Enter name&quot;</code></li>
<li>Numeric attributes: Unquoted numbers <code>min=18</code>, <code>max=100</code></li>
<li>All attributes are passed directly to HTML elements</li>
</ul>
