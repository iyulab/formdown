<li>Full-screen editing experience</li>
</ul>

<ul>
<li>Writing new forms from scratch</li>
<li>Focused editing sessions</li>
<li>When you know the syntax well</li>
<li>Mobile or small screen devices</li>
</ul>

<h3>Recommended Approach</h3>
<ol>
<li><strong>Start with Split Mode</strong> - Begin development using <code>mode=&quot;split&quot;</code> to see immediate visual feedback</li>

<li><strong>Switch to Editor Mode</strong> - Use <code>mode=&quot;editor&quot;</code> for focused editing sessions</li>
<li><strong>Test with View Mode</strong> - Preview final form using <code>mode=&quot;view&quot;</code></li>
<li><strong>Validate and Deploy</strong> - Ensure syntax is correct before production use</li>
</ol>
<h3>Best Practices</h3>
<ul>
<li><strong>Use Split Mode for Learning</strong> - Visual feedback helps understand syntax</li>
<li><strong>Leverage Auto-completion</strong> - Speed up development with built-in suggestions</li>
<li><strong>Enable Validation</strong> - Catch syntax errors early in development</li>
<li><strong>Test Different Devices</strong> - Switch modes based on screen size</li>
<li><strong>Save Frequently</strong> - Enable auto-save for longer editing sessions</li>
</ul>
